💡 Why the Avalanche Effect Matters

CRC32 is designed so that a tiny 1-bit change in input causes roughly half the output bits to change. This is called the Avalanche Effect.

  • If a corrupted superblock accidentally produced the same CRC as the original, the corruption would slip through completely undetected.
  • Because ~50% of output bits flip for any single input bit change, the probability of a false match is only 1 in 4,294,967,296 (1 in 2³²).
  • This is why littlefs trusts CRC32 to guard the superblock — even flash bit-flips or partial writes cannot silently corrupt the metadata.
